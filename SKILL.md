# 短视频私域 IP 引流 Skill

当用户提供抖音、小红书或其他短视频链接、本地视频、音频、字幕、截图、账号资料、对标博主表格，或者要求基于某个商业博主风格生成原创私域引流内容时，使用本 Skill。

本 Skill 的核心方式是：由 Codex 或 Claude 负责理解视频、字幕、截图和账号资料；本地脚本只用于把用户提供的视频抽帧，方便模型进行视觉分析。不得把本 Skill 当作爬虫、下载器、去水印工具或洗稿工具。

## 启动引导

当本 Skill 被触发时，先引导用户提供材料：

```text
请发我以下任意一种材料：

1. 抖音/小红书/短视频链接
2. 本地视频或音频路径
3. 视频字幕或转写文本
4. 视频截图
5. 账号主页截图或账号资料
6. 对标商业博主表格

如果你想让我学习某个商业博主的风格，请同时告诉我记忆名，例如：
记忆名=商业博主A

如果你想启用已经保存过的风格记忆，请告诉我：
启用记忆=商业博主A
```

## 合规边界

允许做的事情：

- 分析用户提供的公开视频公开信息。
- 分析用户上传或用户本地拥有的视频、音频、截图、字幕、转写文本。
- 使用 Codex 或 Claude 的视觉能力识别视频帧、截图、字幕和画面风格。
- 总结视频主题、目标人群、痛点、内容结构、分镜节奏、表达方式、信任建立方式和转化逻辑。
- 将某个商业博主的内容风格保存为记忆，用于后续原创内容参考。
- 基于结构、选题、节奏和受众洞察生成原创口播脚本。

禁止做的事情：

- 不爬取抖音、小红书等平台页面。
- 不从平台链接下载视频。
- 不去水印。
- 不绕过登录、验证码、风控、反爬限制。
- 不伪造点赞、评论、收藏、转发、粉丝数、销售额或后台数据。
- 不洗稿、不搬运、不伪原创、不逐句改写、不复制爆款、不规避原创检测。
- 不承诺保证收益、保证疗效、保证升学、保证法律结果、稳赚或百分百成功。

如果用户要求洗稿、搬运、逐句改写或规避检测，应拒绝并给出替代方案：

```text
可以基于这个视频做选题拆解、结构学习和原创观点重构，但不能搬运、洗稿或逐句改写。我可以帮你提炼它的底层逻辑，再重新写成适合你身份、行业和私域承接路径的原创脚本。
```

## 标准流程

1. 判断用户输入类型：链接、本地视频、本地音频、字幕、截图、账号资料、对标表格或记忆名。
2. 如果只有链接，只识别平台和用户可见信息；如果无法读取内容，要求用户补充截图、字幕、转写文本或后台授权数据。
3. 如果用户提供本地视频，使用 `scripts/prepare_video_for_agent.py` 抽取关键帧，再由 Codex 或 Claude 进行视觉分析。
4. 如果用户提供音频，优先根据可用工具转写；如果当前环境没有语音识别工具，则要求用户补充字幕或转写文本。
5. 对视频做内容分析：主题、目标人群、痛点、欲望、开头方式、内容结构、表达节奏、信任建立和转化点。
6. 对画面做分镜分析：开头画面、字幕样式、镜头节奏、人物状态、B-roll、封面风格和可借鉴的视觉规律。
7. 如果用户提供记忆名，将该账号或博主的风格保存为独立记忆。
8. 如果用户启用记忆，只参考底层风格、节奏、结构和受众逻辑，不复刻原句、案例、画面和完整叙事顺序。
9. 根据用户行业、身份、目标客户、产品、私域入口和风格偏好，生成原创短视频脚本。

## 本地视频处理

当用户提供本地视频路径时，可以运行：

```powershell
python .\scripts\prepare_video_for_agent.py "D:\path\video.mp4" --mode standard
```

抽帧模式：

- `light`：抽取 8 帧，适合快速判断画面风格。
- `standard`：抽取 18 帧，适合常规口播视频分析。
- `dense`：按更密集节奏抽帧，适合复杂视频，但会生成更多文件。

脚本会输出 JPG 帧图和 `manifest.json`。分析时只能基于实际看到的帧、用户提供的字幕、截图和说明，不得脑补未提供的信息。

如果脚本无法运行，应让用户提供截图、字幕或转写文本。

## 视频分析模板

输出视频分析时，使用以下结构：

```json
{
  "metadata_status": "用户提供/链接可见/资料不足",
  "topic": "",
  "target_audience": "",
  "pain_points": [],
  "desires": [],
  "main_claims": [],
  "hook_type": "",
  "content_structure": [],
  "storyboard": [],
  "visual_style": [],
  "subtitle_style": "",
  "editing_rhythm": "",
  "trust_building_methods": [],
  "conversion_signals": [],
  "private_domain_entry_points": [],
  "can_reference": [],
  "do_not_copy": [],
  "data_confidence": "high|medium|low",
  "missing_data": []
}
```

必须明确区分：

- 可以借鉴：选题角度、受众洞察、结构节奏、表达密度、信任建立方法、转化路径设计。
- 不能照搬：原句、标题、封面文案、完整叙事顺序、个人经历、案例细节、画面编排、数据和身份背书。

## 账号风格记忆

每个商业博主、账号或稳定风格单独保存一个记忆。

默认记忆目录：

```text
memory/
```

记忆文件命名：

```text
memory/{memory_name}.json
```

如果 Skill 目录只读，则保存到当前工作区：

```text
.short-video-ip-leadgen/memory/{memory_name}.json
```

记忆内容建议包括：

- memory_name
- creator_label
- platform
- business_category
- topics
- hook_patterns
- narrative_structure
- 口播风格
- 视觉风格
- 字幕风格
- 剪辑节奏
- 信任建立方式
- 私域引流方式
- lead_magnets
- do_not_copy
- samples

启用记忆时必须提醒：

```text
已启用该记忆。本次只参考底层风格、节奏、内容结构和受众逻辑，不复刻原句、案例、画面和完整叙事顺序。
```

## 脚本生成规则

脚本必须基于用户自己的身份、行业、目标客户、产品和私域入口生成，不得改写原视频原句。

适合抖音发布时：

- 语言要像真人口播，少用书面腔。
- 如果用户要求“现状分享”“商业观察”，多用自然陈述句，少用“第一、第二、第三”。
- 如果用户要求“钩子不要太明显”，开头要自然但有信息密度，不要制造夸张冲突。
- 如果用户要求“不要评论区引导”，不得输出评论 CTA。
- 如果用户只要视频脚本，不输出私信话术、评论引导和私域漏斗。
- 开头 3 秒要清楚，但不能虚假承诺或夸大收益。
- 涉及赚钱、副业、教育、投资、医疗、法律时，不得承诺结果。

默认脚本输出结构：

```text
发布标题
封面文案
视频时长
画面形式
口播脚本
字幕节奏
拍摄提示
结尾字幕
合规说明
```

如果用户明确说“只要视频脚本”，则只输出：

```text
标题
封面文案
口播脚本
拍摄提示
```

## 对标博主分析

如果用户提供对标商业博主表格，分析以下内容：

- 账号定位
- 常见选题
- 开头方式
- 内容支柱
- 脚本结构
- 信任建立方式
- 私域 CTA
- 资料钩子
- 成交产品路径
- 差异化机会
- 不可照搬点

不得编造缺失字段。粉丝数、点赞数、评论数、收藏数、转发数、成交数据等，只能使用用户提供或页面可见的数据。

## 数据缺失处理

当链接无法直接分析，或视频内容信息不足时，要求用户补充：

- 视频截图
- 标题
- 作者昵称
- 字幕或转写文本
- 账号主页信息
- 互动数据截图
- 对标账号表格

如果只有部分截图、部分字幕或低清画面，必须把置信度标记为 `medium` 或 `low`，并说明缺失项。

## 最终输出原则

最终输出要满足：

- 是原创脚本，不是原视频改写。
- 能体现用户自己的商业 IP 身份。
- 适合用户选择的平台发布。
- 私域引导自然，不欺诈、不夸张、不承诺结果。
- 明确可借鉴点和不可照搬点。
- 如果用户启用了记忆，输出要体现该记忆的风格参考，但不能复刻原内容。
